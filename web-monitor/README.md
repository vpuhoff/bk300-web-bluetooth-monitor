# BK300 Voltage Monitor (Web Bluetooth)

Веб‑приложение для подключения к BK300 по BLE и отображения **напряжения**
в реальном времени. UI в стиле shadcn, график — uPlot.

## Требования

- Браузер с Web Bluetooth: Chrome / Edge.
- Контекст **HTTPS** (или `http://localhost`).

## Запуск

```bash
cd web-monitor
npm install
npm run dev
```

Откройте страницу, нажмите **«Подключить BK300»** и выберите устройство `BK300`.

## Что читает приложение

- Команда **`0B0B`** (раз в секунду) → ответ **`4B0B`**
- Напряжение: первые 2 байта payload как **U16 little-endian**, \(V = U16 / 100.0\)

На старте также отправляется инициализация (kick), близкая к Android-клиенту:

```
0301 → 0204 → 0100
```

> ⚠ Полный «канонический» kick из HCI Android-клиента включает ещё `0B01`
> (синхронизация часов прибора с Unix-epoch браузера). Web-вариант его не
> отправляет, но `Date.now() / 1000` всё равно валидное число, так что
> часть прошивок BK300 нормально работает и без него. Если конкретно у
> тебя web-monitor не выводит данные — попробуй добавить `0B 01` с
> текущим Unix-epoch перед `0301` (см. `docs/spec.md`, раздел «КРИТИЧНО:
> `0B01` ожидает реальный Unix-epoch»).

## Автовосстановление

При обрыве `gattserverdisconnected` запускается реконнект с экспоненциальной
задержкой и повторной инициализацией.

## См. также

- `docs/spec.md` — спецификация BLE-протокола BK300 (формат кадра, CRC,
  команды, режим waveform).
- `esp32-idf/bk300_monitor/` — нативная реализация на ESP32-S3 + ESP-IDF
  Bluedroid; полезно сравнить, если нужно отладить web-вариант.

## Технологический стек

React + TypeScript + Vite, Tailwind / shadcn-ui, uPlot, Web Bluetooth API.
